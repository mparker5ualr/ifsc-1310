About This Assignment

This page demonstrates a refactor from older, generic HTML structure into modern HTML5 semantic markup. Instead of relying only on generic div elements, this version uses meaningful elements that describe the purpose of each section of the page.

HTML5 Semantic Structure

Header

The <header> element is used for introductory content such as the page title and brief description.

Navigation

The <nav> element contains the main navigation links for the page.

Main Content

The <main> element identifies the primary content of the page.

Sections and Articles

The <section> element groups related content, while the <article> element represents content that can stand on its own.

Footer

The <footer> element contains closing information such as copyright and related links.

Why Semantics Matter

Semantic HTML improves accessibility, search engine understanding, and code readability. It helps browsers, screen readers, and developers understand the purpose of the content.

Reflection

Refactoring HTML into semantic elements makes the structure of a web page clearer. This helps create a stronger foundation for styling, responsive design, accessibility, and future maintenance.
